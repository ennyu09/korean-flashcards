## 项目是什么

一个韩语单词闪卡自学网站，个人项目，非商业用途，会发到小红书给别人免费用。
核心功能：翻卡片学词汇 + 印章式记忆分级（모르겠어요/조금 알아요/거의 알아요/완벽해요）
+ 间隔复习 + 语法分析（含跳转到语法速查页）+ 预生成发音（带语速调节）。

部署：GitHub（`ennyu09/korean-flashcards`，public）+ Vercel 自动构建，
上线地址 https://korean-flashcards-brown.vercel.app/

## 数据结构（每条词汇的字段，vocab.json 里用短字段名）

```json
{
  "k": "국적",
  "r": "gukjeok",
  "p": "名词",
  "m": "国籍",
  "e": "제 국적은 중국입니다.",
  "t": "我的国籍是中国。",
  "g": [
    {"text": "제", "meaning": "我的", "note": "저(我，敬语)+의(所有格助词)的缩略形式"},
    {"text": "국적", "meaning": "国籍", "note": "名词", "isTarget": true},
    {"text": "은", "meaning": "—", "note": "主题助词，标记句子的主题"},
    {"text": "중국", "meaning": "中国", "note": "名词"},
    {"text": "입니다", "meaning": "是……", "note": "이다(是)的敬语终结形：词干+ -ㅂ니다"}
  ],
  "st": "陈述句",
  "fl": "尊敬阶（합니다体）"
}
```

字段规则（incoming 文件里用全称，导入时转换成短字段——对应关系见下方"文件交接工作流"）：
- `m`（meaning）：只放纯词义，**不要**塞括号里的敬语/语体标注（那是 `rg` 字段的事）
- `rg`（register）：敬阶信息单独放这里，值如"中立"、"敬语"、"非敬语（반말）"，
  只有不是"中立"才写进 vocab.json（中立是默认值，省略不写），卡片上也只有不是中立才显示小标签
- `p`（pos）：显示时用【】包起来，如"【名词】"
- `g`（grammarBreakdown）：逐词/逐助词拆解，每个元素是 text/meaning/note 三个字段，
  纯语法助词没有独立词义的，meaning 写"—"。默认折叠显示，点击展开成表格
  - `isTarget: true`：标在这条卡片正在教的那个词（可能是变形后的样子）对应的那一项上，
    前端渲染时会加粗/变色高亮这一行，让学习者一眼看出"这个是我在学的词，其他是辅助内容"
  - note 里如果出现语法速查表收录的简写（比如"-습니다"），前端会自动识别并做成可点击的
    跳转链接，不需要手动加任何标记，识别规则表见 `문법_링크_매핑.json`
- `hi`（homonymIndex）：官方词典里的同形异义词编号，只在某个词的拼写跟其他词形完全一样
  （不管是不是在同一个Unit里）时才带这个字段，取自官方词典编号，不用自己判断哪些词算重复。
  渲染成卡片正面单词右上角的小号上标数字。**跨Unit的同形词如果读音相同，发音音频文件本身
  可以共用，但例句音频文件名要带上 hi 后缀区分**（脚本已处理，见 `scripts/generate-audio.js`）
- `st`（sentenceType）：陈述句/疑问句/命令句（请求）/共动句
- `fl`（formalityLevel）：尊敬阶（합니다体）/泛尊阶（해요体）/非敬语（반말）
  （旧说法"格式体敬语/非格式体敬语"已在 v2 批量更新中统一改成"尊敬阶/泛尊阶"，
  括号里的합니다体/해요体传统语法名称保留不变）
- 不规则动词/形容词：凡例句中出现 ㅂ/ㄷ/ㅅ/르/ㅎ 不规则活用或 으 脱落、ㄹ脱落 且实际改变了
  表面形态的，note 字段里要写清楚具体是哪类不规则、怎么变化，不能只写"词干变化"这种模糊说法

## 罗马音生成（`scripts/regenerate-romanization.js`）

用 [koroman](https://github.com/gerosyab/koroman) 库依据国立国语院《国语的罗马字表记法》
批量生成，**不能盲信库的输出**——已实测发现 koroman 在下面几类场景有系统性 bug，
脚本里用 `ROMANIZATION_OVERRIDES` 手动订正（每条都有注释写清楚依据），
以后遇到新词触发同样的坑，照这个思路查国立국어원온라인가나다问答验证再加override：

- **받침(ㅅ→代表音ㄷ) + 하 送气音合并成ㅌ时**，koroman 错误保留字母h（比如
  깨끗하다错误输出成kkaekkeuthada），但官方标准送气音合并后只用合并后辅音本身的罗马字母
  （参照 좋고→joko、낳다→nota、산책하다→sanchaekada 都不保留h），应该是 -tada 不是 -thada
- **合成词ㄴ添加规则（사잇소리현상）判断不可靠**，这个规则本身是词典层面的、不能从拼写
  机械推导，koroman 有时会多加（목요일→错误输出mongnyoil，实际只是连音[모교일]），
  有时会漏加（집안일→错误漏掉ㄴ添加，实际发音是[지반닐]）。每次批量生成后**必须**把
  diff 报告里的每个变化词过一遍国立국어원在线问答核实，不能只看"库输出了就是对的"
- **맛없다**类词（받침ㅅ按代表音处理后再连音，而不是直接连音）：국립국어원问答明确
  맛없다只有[마덥따]一个标准发音（不像"맛있다"允许两种读法）

## 文件交接工作流

- 用户在自己的对话里让另一个Claude实例生成新Unit的json文件，文件名格式
  `unitXX_주제韩语名.json`，XX是两位数编号，字段用全称：
  `korean/romanization/pos/meaning/example/exampleTranslation/grammarBreakdown/register/sentenceType/formalityLevel/homonymIndex`
  （对应vocab.json里的 `k/r/p/m/e/t/g/rg/st/fl/hi`，导入时自动转换，grammarBreakdown数组
  里的 `text/meaning/note/isTarget` 字段名不变）
- 用户下载后会放进项目里的 `/incoming` 文件夹，有时还会带一份更新过的 `CLAUDE.md`
  （记录这一批内容的最新权威结构/进度），看到就要通读，可能需要用它来更新这份文件本身
- 看到 `/incoming` 里有新文件时：**完整替换**对应Unit的全部数据（所有字段），
  不要只更新部分字段（之前出过"只换了词，没加上词性"的问题，切记要全量替换）。
  如果是**整个课程目录结构重建**（Unit编号/Chapter归属大规模变动，不是零星加几课），
  要以用户提供的最新结构说明为唯一准确来源，整个替换掉旧菜单结构，不要在旧结构上小修小补
- 替换后要抽查至少5-8个词（结构大改时抽查范围要覆盖不同Chapter），确认每个字段都真的
  显示出来了，不要只测第一条就当没问题
- 每次导入完，都要重跑 `npm run romanize`（全量重算，免费、瞬间完成，不用心疼）和
  `npm run audio`（增量执行，只给还没有音频文件的词补，不会浪费额度重新生成已有的）
- 处理完的incoming源文件移进 `incoming/imported/` 子文件夹（没有就新建），保持
  `/incoming` 根目录只留"还没处理"的文件
