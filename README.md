# 盖章背单词 · 韩语单词闪卡

一个纯静态的韩语单词闪卡网页：翻卡片 + 印章式记忆分级 + 间隔重复复习 + 按 Unit 分组的目录。

## 复习间隔怎么算的

设计目标是"每天打开一次就够，不用一天点好几次"，所以间隔从 1 天起步，答对就大约翻倍拉长：**1天 → 2天 → 4天 → 7天 → 15天 → 30天 → 60天**（共 7 级）。

- 点「完全不会」：不走上面的长阶梯，**10分钟后**再考你一次——真忘了的词等一整天才复习等于白学，趁还没完全忘掉赶紧再看一眼才有效；但这不强迫你当天必须打开app，只是那天如果还在用会再冒出来
- 点「有点印象」：停在当前这一步，不前进也不后退
- 点「比较熟悉」：往后推进一步
- 点「完全掌握」：一次推进两步（跳得更快）

到了"60天"这一步之后就不再继续拉长间隔了，每次复习对都停在60天后再考，算是长期记住了。

## 文件说明

- `index.html` — 页面结构、样式和全部逻辑代码。
- `vocab.json` — 单词数据，按"等级 → 章节 → 单元"三层组织。新增/修改单词只需要编辑这个文件，不用碰 `index.html`。

## vocab.json 结构

```json
{
  "levels": [
    {
      "id": "beginner",
      "labelKo": "초급",
      "labelZh": "初级",
      "status": "active",
      "chapters": [
        {
          "title": "Chapter 1 · 从零开始",
          "units": [
            { "id": 1, "title": "打招呼与自我介绍", "words": [ ... ] },
            { "id": 2, "title": "数字与基础时间表达", "words": [ ... ] },
            { "id": 3, "title": "核心动词①", "words": [] }
          ]
        }
      ]
    },
    { "id": "intermediate", "labelKo": "중급", "labelZh": "中级", "status": "coming_soon", "chapters": [] },
    { "id": "advanced", "labelKo": "고급", "labelZh": "高级", "status": "coming_soon", "chapters": [] }
  ]
}
```

- 三个等级固定是初/中/高级，`status` 是 `"active"`（正常使用）或 `"coming_soon"`（首页显示"内容制作中"占位，Chapter 数组留空）。
- 初级目前有 9 个 Chapter、37 个 Unit，`words` 是空数组的 Unit 会在首页自动显示"待更新"标签、点了提示"内容制作中"，不会报错，只要往里面填了词就自动变成可点击。
- Unit 的 `id` **全局唯一**（1～37 不重复），换单元不用担心编号冲突。

## 单词字段说明

```json
{
  "k": "감사",
  "r": "gamsa",
  "p": "名词",
  "m": "感谢",
  "e": "도와주셔서 감사합니다.",
  "t": "谢谢您的帮助。",
  "g": [
    {"x":"도와주셔서", "y":"谓语(原因状语)：도와주다(帮忙)+ -시-(尊敬) + -어서(因为...)"},
    {"x":"감사합니다", "y":"谓语：감사(感谢)+하다+ -ㅂ니다(敬语终结词尾)"}
  ],
  "rg": "非敬语（반말）"
}
```

| 字段 | 含义 | 是否必填 |
|---|---|---|
| `k` | 韩语单词 | 必填 |
| `r` | 罗马音标注 | 必填 |
| `p` | 词性（名词/动词/形容词……） | 选填，不填就不显示词性标签 |
| `m` | 中文释义 | 必填 |
| `e` | 韩语例句 | 必填 |
| `t` | 例句中文翻译 | 必填 |
| `g` | 例句成分拆解，数组，每项 `{x:词/短语, y:语法说明}` | 选填，不填就不显示"成分拆解"区块 |
| `rg` | 敬阶标注（比如"非敬语（반말）"），只在不是中立/默认用法时填 | 选填，不填就不显示红色标签 |

## 怎么加新 Unit 的词

找到 `vocab.json` 里对应 Unit 的 `words` 数组（空的 `[]`），仿照上面的格式把词填进去即可，存好刷新网页就能看到，首页的"待更新"标签会自动消失变成可点。

## 新 Unit 文件命名约定

以后每个 Unit 的原始词表文件统一按 `unitXX_韩语主题名.json` 命名（比如 `unit03_핵심동사1.json`），数组里每项用这几个字段：`korean / romanization / pos / meaning / example / exampleTranslation / grammarBreakdown / register`（跟上面 `k/r/p/m/e/t/g/rg` 一一对应，我导入时会自动转换）。发文件给我时按这个格式来，方便对应到正确的 Chapter/Unit，不容易搞混。

## 学习进度保存在哪里

学习进度和"今天已学新词数"（仅做统计展示，不限制数量）都保存在**访问者自己浏览器的 localStorage** 里，不会上传到任何服务器，我也看不到。换浏览器或换设备进度不会同步，清除浏览器数据（或用无痕/隐私模式）会导致进度丢失。到期复习的词和没学过的新词会一起出现，没有每日数量限制。

## 发音怎么来的 / 怎么替换某个词的发音

"🔊 播放发音"默认用访问者浏览器自带的语音朗读引擎（Web Speech API），不需要额外文件，但音色/准确度取决于访问者的设备，没法保证所有人听到的都一样、都准。

如果发现某个词读得不对，可以单独给它换成一份预先生成好的音频文件（不影响其他词）：

1. 用免费 TTS 网站（比如 TTSMaker）生成这个词的语音，下载 mp3
2. 把文件放进项目里的 `audio/` 文件夹
3. 在 `index.html` 里找到 `AUDIO_OVERRIDES` 这个对象，加一行，比如：
   ```js
   const AUDIO_OVERRIDES = {
     "좀": "audio/좀.mp3",
     "좀_例句": "audio/좀_例句.mp3"   // 例句发音用"单词_例句"作为 key
   };
   ```
4. 存好、`git push`，这个词以后就播放这份文件了，其他没加的词继续用浏览器语音

这个替换是逐词可选的，不需要一次性把所有词都换掉。

## 报错按钮怎么工作的

卡片背面左上角的"🐛"按钮点了会打开访问者自己的邮件客户端，收件人固定是 yikenyu@gmail.com，标题和正文自动带上当前这个词的单元/单词/例句信息。因为是纯静态网站没有后端，这是零成本能做到的"报错反馈"方式；如果访问者的设备没配置默认邮件客户端，点了可能没反应，这个目前没有更好的替代方案。

## 本地怎么预览修改效果

因为页面用 `fetch` 读取 `vocab.json`，不能直接双击 `index.html` 打开（会因为浏览器的本地文件限制加载失败），需要用一个本地小型服务器打开，例如在这个文件夹里执行：

```
python3 -m http.server 5500
```

然后浏览器打开 `http://localhost:5500` 查看。

## 部署

这是纯静态网站（没有后端），推送到 GitHub 后可以直接用 Vercel / Netlify / GitHub Pages 等任意静态托管平台部署。当前部署在 Vercel，`git push` 到 `main` 分支后会自动重新部署。
